# 06 — Tool Error Handling

> *Production agents break in the real world — robust error handling is what separates demos from deployable systems.*

---

## 6.1 Why Error Handling Is Critical for Agents

A tool call in an agent is not a simple function call. It involves:
- External APIs that can be down
- Rate limits that can be hit
- Network timeouts
- Invalid arguments generated by the LLM
- Partial results or empty responses
- Authentication failures

Without proper error handling:
- One tool failure crashes the **entire** multi-step agent run
- The LLM gets confused when tool results are missing
- Costs spike from retrying with expensive models
- Users get cryptic error messages instead of graceful degradation

## 6.2 Error Classification

Categorize errors before handling them:

```python
from enum import Enum

class ToolErrorType(Enum):
    # Transient — worth retrying
    RATE_LIMIT = "rate_limit"           # Too many requests, back off and retry
    TIMEOUT = "timeout"                 # Slow network/API, retry with longer timeout
    SERVICE_UNAVAILABLE = "service_unavailable"  # 503, retry after delay
    
    # Permanent — don't retry, handle differently
    NOT_FOUND = "not_found"             # 404 — resource doesn't exist
    INVALID_ARGS = "invalid_args"       # Bad arguments from LLM — need better schema
    AUTH_FAILED = "auth_failed"         # 401/403 — check credentials
    
    # Partial success — use what you have
    PARTIAL_RESULT = "partial_result"   # Got some data but not all
    EMPTY_RESULT = "empty_result"       # Succeeded but found nothing

def classify_error(exception: Exception, status_code: int = None) -> ToolErrorType:
    if status_code == 429:
        return ToolErrorType.RATE_LIMIT
    elif status_code == 404:
        return ToolErrorType.NOT_FOUND
    elif status_code in (401, 403):
        return ToolErrorType.AUTH_FAILED
    elif status_code == 503:
        return ToolErrorType.SERVICE_UNAVAILABLE
    elif isinstance(exception, TimeoutError):
        return ToolErrorType.TIMEOUT
    elif isinstance(exception, ValueError):
        return ToolErrorType.INVALID_ARGS
    else:
        return ToolErrorType.SERVICE_UNAVAILABLE
```

## 6.3 Retry Patterns

### Pattern 1: Simple Retry with Fixed Delay

```python
import time

def retry_with_delay(func, args, max_retries=3, delay_seconds=2):
    for attempt in range(max_retries):
        try:
            return func(**args)
        except Exception as e:
            if attempt < max_retries - 1:
                print(f"Attempt {attempt+1} failed: {e}. Retrying in {delay_seconds}s...")
                time.sleep(delay_seconds)
            else:
                raise
```

### Pattern 2: Exponential Backoff (Production Standard)

```python
import time
import random

def retry_with_exponential_backoff(
    func,
    args: dict,
    max_retries: int = 4,
    base_delay: float = 1.0,
    max_delay: float = 60.0,
    jitter: bool = True
) -> str:
    """
    Retry with exponential backoff + jitter.
    Delay: base * 2^attempt (+ random jitter to prevent thundering herd)
    """
    for attempt in range(max_retries):
        try:
            return func(**args)
        
        except RateLimitError:
            if attempt >= max_retries - 1:
                return "Error: Rate limit exceeded after all retries."
            
            delay = min(base_delay * (2 ** attempt), max_delay)
            if jitter:
                delay += random.uniform(0, delay * 0.1)
            
            print(f"Rate limited. Waiting {delay:.1f}s before retry {attempt+1}...")
            time.sleep(delay)
        
        except TimeoutError:
            if attempt >= max_retries - 1:
                return "Error: Request timed out after all retries."
            print(f"Timeout on attempt {attempt+1}. Retrying...")
            time.sleep(base_delay)
        
        except (AuthError, NotFoundError) as e:
            # Permanent errors — don't retry
            return f"Error (not retrying): {str(e)}"
        
        except Exception as e:
            if attempt >= max_retries - 1:
                return f"Error after {max_retries} retries: {str(e)}"
            time.sleep(base_delay)
    
    return "Error: Max retries exceeded"
```

## 6.4 The Error Result Pattern — Telling the LLM What Happened

When a tool fails, **don't raise an exception that crashes the loop**. Instead, return a descriptive error string so the LLM can adapt:

```python
# ❌ BAD — crashes the entire agent loop
def call_tool(name, args):
    result = external_api_call(args)  # raises if fails
    return result

# ✅ GOOD — returns error message so LLM can handle gracefully
def call_tool(name, args) -> str:
    try:
        result = external_api_call(args)
        return str(result)
    except RateLimitError:
        return "ERROR_RATE_LIMIT: The search API is temporarily rate-limited. Try a different query or wait 30 seconds."
    except TimeoutError:
        return "ERROR_TIMEOUT: The API call timed out after 30 seconds. The service may be slow."
    except ValueError as e:
        return f"ERROR_INVALID_INPUT: {str(e)}. Please provide a valid input and try again."
    except Exception as e:
        return f"ERROR_UNKNOWN: Tool call failed with: {str(e)}"
```

The LLM then reads this error result and can:
- Try a different query
- Skip this tool and use another
- Report the failure to the user
- Ask for alternative input

## 6.5 Graceful Degradation — Always Have a Fallback

```python
async def web_search_with_fallback(query: str) -> str:
    """Try primary search, fall back to alternatives."""
    
    # Try 1: Primary search API (Tavily)
    try:
        result = await tavily_search(query)
        if result and len(result) > 50:
            return result
    except Exception as e:
        print(f"Tavily failed: {e}")
    
    # Try 2: Backup search API (SerpAPI)
    try:
        result = await serpapi_search(query)
        if result and len(result) > 50:
            return result
    except Exception as e:
        print(f"SerpAPI failed: {e}")
    
    # Try 3: DuckDuckGo (free, no API key)
    try:
        result = await duckduckgo_search(query)
        return result
    except Exception:
        pass
    
    # All failed — return honest error for LLM to handle
    return f"ERROR: All search providers unavailable for query '{query}'. Please answer from training knowledge or ask user for the information."
```

## 6.6 Timeout Management

Always set timeouts — never let tool calls run indefinitely:

```python
import asyncio

async def execute_with_timeout(
    tool_func,
    args: dict,
    timeout_seconds: float = 30.0
) -> str:
    try:
        result = await asyncio.wait_for(
            asyncio.coroutine(tool_func)(**args),
            timeout=timeout_seconds
        )
        return str(result)
    except asyncio.TimeoutError:
        return f"ERROR_TIMEOUT: Tool took longer than {timeout_seconds}s. Try a simpler query."

# Per-tool timeout configuration
TOOL_TIMEOUTS = {
    "web_search": 15,          # Fast search
    "read_url": 30,            # Web page load
    "execute_code": 60,        # Code execution
    "database_query": 10,      # Should be fast
    "send_email": 20           # Network + SMTP
}

async def safe_execute(tool_name: str, args: dict) -> str:
    timeout = TOOL_TIMEOUTS.get(tool_name, 30)
    return await execute_with_timeout(TOOL_MAP[tool_name], args, timeout)
```

## 6.7 Result Validation — Don't Trust Tool Outputs Blindly

Just as you validate LLM output, validate tool results before passing back:

```python
def validate_and_clean_result(tool_name: str, raw_result: str) -> str:
    """Validate and sanitize tool results before injecting into LLM context."""
    
    # Check for empty results
    if not raw_result or len(raw_result.strip()) < 5:
        return f"EMPTY_RESULT: {tool_name} returned no useful data."
    
    # Truncate overly long results (protect context window)
    MAX_RESULT_TOKENS = 2000  # ~8000 chars
    if len(raw_result) > 8000:
        raw_result = raw_result[:8000] + "\n[...result truncated to fit context window]"
    
    # Remove common injection patterns from tool output (security)
    injection_patterns = ["ignore previous instructions", "you are now", "new system prompt"]
    for pattern in injection_patterns:
        if pattern.lower() in raw_result.lower():
            raw_result = f"[SECURITY: Potential injection in tool result filtered]\n{raw_result}"
            break
    
    return raw_result
```

## 6.8 Circuit Breaker Pattern

For high-frequency agent systems, a circuit breaker prevents repeatedly calling a known-failed tool:

```python
class CircuitBreaker:
    """
    3 states: CLOSED (normal) → OPEN (failing) → HALF_OPEN (testing recovery)
    After N failures → OPEN (don't call tool for cooldown_seconds)
    After cooldown → HALF_OPEN (try once)
    If success → CLOSED again
    """
    def __init__(self, failure_threshold=3, cooldown_seconds=60):
        self.failure_count = 0
        self.failure_threshold = failure_threshold
        self.cooldown_seconds = cooldown_seconds
        self.last_failure_time = None
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
    
    def can_call(self) -> bool:
        if self.state == "CLOSED":
            return True
        elif self.state == "OPEN":
            # Check if cooldown has passed
            if time.time() - self.last_failure_time > self.cooldown_seconds:
                self.state = "HALF_OPEN"
                return True
            return False
        elif self.state == "HALF_OPEN":
            return True
    
    def record_success(self):
        self.failure_count = 0
        self.state = "CLOSED"
    
    def record_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        if self.failure_count >= self.failure_threshold:
            self.state = "OPEN"
            print(f"Circuit breaker OPEN: tool disabled for {self.cooldown_seconds}s")

# Usage
breakers = {tool: CircuitBreaker() for tool in TOOL_MAP}

def execute_tool_with_breaker(tool_name: str, args: dict) -> str:
    breaker = breakers[tool_name]
    
    if not breaker.can_call():
        return f"ERROR_CIRCUIT_OPEN: {tool_name} is temporarily disabled due to repeated failures. Try again later."
    
    try:
        result = TOOL_MAP[tool_name](**args)
        breaker.record_success()
        return result
    except Exception as e:
        breaker.record_failure()
        return f"ERROR: {str(e)}"
```

## 6.9 Error Logging — Observability for Debugging

```python
import logging
from datetime import datetime

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger("agent.tools")

def execute_with_logging(tool_name: str, args: dict) -> dict:
    """Execute tool with structured logging."""
    start_time = datetime.now()
    
    try:
        result = TOOL_MAP[tool_name](**args)
        duration_ms = (datetime.now() - start_time).total_seconds() * 1000
        
        logger.info({
            "event": "tool_success",
            "tool": tool_name,
            "args": args,
            "duration_ms": round(duration_ms, 2),
            "result_length": len(str(result))
        })
        return {"status": "success", "result": result}
    
    except Exception as e:
        duration_ms = (datetime.now() - start_time).total_seconds() * 1000
        
        logger.error({
            "event": "tool_error",
            "tool": tool_name,
            "args": args,
            "error_type": type(e).__name__,
            "error_msg": str(e),
            "duration_ms": round(duration_ms, 2)
        })
        return {"status": "error", "error": str(e)}
```

## 📌 Key Takeaways

1. **Classify errors** — transient (retry) vs permanent (handle) vs partial (use what you have)
2. **Exponential backoff** — standard pattern for rate limits and transient failures
3. **Return error strings, not exceptions** — keep the agent loop running, let the LLM adapt
4. **Always set timeouts** — never let a tool call block indefinitely
5. **Validate results** — truncate long outputs, detect empty results, sanitize injections
6. **Circuit breaker** — stop calling repeatedly-failing tools; open the circuit temporarily
7. **Log everything** — tool name, args, duration, result length — critical for debugging production agents
